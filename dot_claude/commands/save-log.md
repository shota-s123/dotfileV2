# セッションログ保存

現在のセッションの作業ログを Obsidian vault の Inbox に保存する。
Inbox は一時置き場。あとで `/inbox-processor` が PARA に振り分ける。

## 保存先

`~/workspace/my-kb-vault/00_Inbox/`（環境変数 `OBSIDIAN_VAULT` が設定されていればそちらを優先）

## 手順

1. セッション内容を振り返る
2. 下記フォーマットでノートを作成・保存
3. ファイルパスを報告

## ファイル名

`YYYY-MM-DD - [作業内容の要約].md`

## フォーマット

```markdown
---
tags:
  - ログ
  - [関連トピック1]
  - [関連トピック2]
created: YYYY-MM-DDTHH:mm:ss+09:00
source: claude-code
---

[2〜3文で何をしたか。背景・動機・現在の状況も含める]

## 作業メモ

セッション中に調査・議論・実施した内容をトピックごとにサブセクションに分けて記録する。

### [トピック1]
- 具体的な事実・数字・料金・URL等を含めて記録
- 比較検討した場合は表形式で整理する
- 選択肢ごとのメリット・デメリットを明記

### [トピック2]
- 同上

## 学んだこと
- 知識、発見、気づき（あとで 03_Resources に移動される想定）
- 具体的な数字やデータを含める（シェア、料金、期限など）
- 「なぜそうなのか」の理由も書く

## 次にやること
- [ ] アクション項目は具体的に（URL、手順、必要なものを添える）
```

## 注意事項

- セッションの内容を**できるだけ具体的に**記録する（数字・URL・比較表など）
- トピックが複数ある場合はサブセクションで整理する
- セクションは中身がなければ省略してよい
- 既存ノートとの関連があれば `[[WikiLinks]]` でリンク
- vault 外から呼ばれても必ず上記パスに保存
