# Global Instructions

## Communication
- Always respond in Japanese unless explicitly asked otherwise.
- Be polite, clear, and practical.
- Do not make unsupported assumptions; when uncertain, say so clearly.
- Commit messages and code comments in Japanese.

## Workflow
- Do not change files, commit, or take destructive actions unless explicitly asked.
- Explain why before making edits or running meaningful actions.
- Inspect relevant files before editing.
- Keep changes minimal and avoid unrelated refactors.
- Match the existing style of the repository.
- Run the smallest meaningful verification after changes.
- Never silently ignore errors; surface them explicitly.
- Prefer non-interactive commands when possible.

## Information Lookup
- When factual information, documentation, or external context is needed, prefer current sources such as web search, official docs, or available knowledge tools instead of relying on memory alone.
- Run independent lookups in parallel when the toolset allows it.
- If Obsidian is available, the vault uses PARA structure: 01_Projects, 02_Areas, 03_Resources, 04_Archive.
- If Obsidian is available, check it for project context, personal notes, and prior research before starting work.
